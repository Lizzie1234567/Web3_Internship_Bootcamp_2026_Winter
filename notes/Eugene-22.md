# Web3 实习手册 - 入门导读

## 区块链基础

### 区块链基础概念

-   去中心化的分布式账本技术
    
-   区块
    
    -   交易记录 + 之前的区块摘要（哈希）
        
-   链
    
    -   串联区块
        

### 如何确保历史记录不可篡改？

-   数据结构（哈希链）：篡改可检测
    
-   分布式网络共识机制：重写数据需要付出巨大成本
    

### 区块链的分类

根据访问权限与治理模式

-   公联
    
    -   自由进出成为节点
        
    -   行为透明化
        
-   联盟链
    
    -   受邀请或申请成为节点
        
    -   权限分为决策者（读写）和观察者（只读）
        
    -   行为半透明
        
-   私链
    
    -   成为节点审批严格，成员固定
        
    -   透明度低，但隐私性强
        

### Web3 核心组成

钱包 → Dapp → 智能合约 → 区块链

-   钱包（MetaMask 等）：管理私钥、发起签名、把 DApp 的请求发送到区块链网络以及确认交易。
    
-   Web 3.0 开发
    
    -   Python + RDFLib + SPARQL
        

## 以太坊概览

### 智能合约

-   存储在区块链上的可执行代码
    
    -   自动执行
        
    -   使用以太币（ETH）来支付相关费用
        

### **权益证明（PoS）机制**

-   质押 32 ETH 成为验证者
    
-   奖励机制、惩罚机制
    

### 架构组成

-   Layer 1，以太坊主网
    
-   Layer 2，二层扩展解决方案
    
-   侧链
    
-   以太坊生态分层架构
    

### 以太坊核心机制

-   以太坊账户分两类
    
    -   外部账户（EOA）：由私钥控制，用于发起交易
        
    -   合约账户（CA）：由合约代码控制，不能主动发起交易，只能被 EOA 调用触发执行
        
-   典型交互路径
    
    -   EOA ↔ EOA：转账
        
    -   EOA → CA：调用合约（如买 NFT、授权 Token、链上投票）
        

### Gas 模型

-   费用计算
    
    -   总费用 = Gas Used × Gas 单价
        
-   Gas 的作用
    
    -   经济激励：让验证者优先打包（给 tip 更容易被处理）
        
    -   防滥用：限制恶意消耗资源（Gas 用完则执行回滚）
        
-   EIP-1559 之后的费用结构
    
    -   Base Fee 基础费：协议自动调整，直接销毁
        
    -   Tip 小费：额外给验证者，用于加速确认
        

### 以太坊虚拟机 EVM

-   核心特点
    
    -   确定性执行：同样输入 → 全网得到同样结果，保证可验证
        
    -   隔离安全：合约运行在沙盒环境，不能随意访问外部系统
        

### 一般流程

-   钱包（EOA）发起交易 → 签名
    
-   交易进入网络 → 验证者打包
    
-   节点在 EVM 执行合约 → 消耗 Gas → 更新状态
    
-   结果写入区块链，可追溯记录
